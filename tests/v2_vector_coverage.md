# v2 -> v3 Test Vector Coverage Survey

Generated by Plan 5 Task 3. Confirms that pyModeS v3's unit-test suite
(981 tests at Plan 5 start, 986 after this task) substantially covers
every positive hex vector pyModeS v2.21.1's test suite used.

## Method

1. Cloned `pyModeS` at tag `v2.21.1` to `/tmp/pyModeS-v2-scratch`.
2. Grepped Mode-S hex literals (14 or 28 hex chars) from v2's
   `tests/test_*.py` sources, yielding 58 unique hex strings.
3. For each hex, checked whether any v3 `tests/test_*.py` (excluding
   `__pycache__`, `data/`, and `fixtures/` so stale bytecode and raw
   sample CSVs do not mask real gaps) references it.
4. Classified any misses by DF / TC / BDS using pyModeS 2.21.1's
   `common.df` and the v2 test file the hex came from.
5. Live-coverage gaps below 10 were filled with minimal positive tests
   in the appropriate v3 test file.

## Results

- Total unique v2 hex vectors surveyed: **58**
- Already covered by v3 tests: **28**
- Missing (all categories): **30**
  - Commented-out in v2 (never live, NIC candidates): **12**
  - Covered by equivalence (same code path, different vector in v3): **17**
  - Genuine coverage gaps filled by this task: **5** (4 surv/allcall + 1 surface-position pair) -- well under the cap of 10.

## Known deferred gaps

- **DF11 II/SI interrogator decode:** pyModeS v2's
  `allcall.interrogator()` emits labels like `"SI6"`; v3 defers full
  II/SI disambiguation to a future plan (the hex `5D484FDEA248F5`
  from v2 is now anchored in
  `tests/test_allcall.py::TestV2VectorSurvey` without asserting
  on the interrogator field). See the DF11 trust-set comment in
  `src/pyModeS/_pipe.py` for the rationale behind keeping DF11
  outside PipeDecoder's trusted-set promotion.

- **ADS-B NIC(v1) from type-code:** v2's `tests/test_adsb.py` carries a
  `def test_nic(): ...` commented out in the source. The 12 hex
  vectors inside it map TC to NIC values 0-11. v3 currently tests the
  `TC_NICv1_lookup` / `TC_NICv2_lookup` tables directly in
  `tests/test_uncertainty_smoke.py` and end-to-end NIC emission is
  not yet wired into `pyModeS.decode()`. Because these v2 tests were
  commented out they are not a regression surface; adding them is
  left for a future plan when the full uncertainty-field pipeline is
  wired in.

- **Sample-CSV batch runs:** pyModeS v2 ships
  `tests/data/sample_data_adsb.csv` (2000 rows) and two
  `sample_data_commb_df{20,21}.csv` files (5000 rows each). v2
  exercises these only through `tests/sample_run_*.py` smoke / timing
  scripts, not pytest. v3 carries the same CSVs under
  `tests/data/` but does not (yet) wire up a batch smoke test.
  `tests/test_batch.py` and `tests/test_pipe.py` already exercise
  batch decoding on small inline lists, so the code path is covered;
  a full CSV replay would be a performance / soak test rather than
  a unit test gap.
